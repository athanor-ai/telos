# Kernel BBRv3 replay — container + sweep

**Status:** live. Two paths:

- `Dockerfile` + `run_sweep.py` — mahimahi + ack_schedule.py;
  lightweight host-kernel sweep harness.
- `sanity_cell.py` + `results/` — sanity-cell harness that boots a
  BBRv3-patched Linux kernel under QEMU/KVM and measures pacing
  collapse on a 5-cell (B, D) grid. The `results/summary.md`
  write-up backs §V.F of the paper.

## What this measures

Real Linux `tcp_bbr.c` (BBRv3 out-of-tree patch) starvation under
clocked-ACK aggregation schedules generated by `ack_schedule.py`.
For each (B, D, link_rate, seed) cell we record:

- `onset_kernel_ms`: first timestamp at which `ss -i` reports
  `pacing_rate` below 1 Kbps.
- `onset_analytic_ms`: the paper's bound $T(B, D)$.
- `residual_ms = onset_kernel − onset_analytic`.

If residuals are bounded by a small constant (say `5·W` ≈ 50ms at
the default W), the abstraction faithfully tracks the deployed
kernel; otherwise we have a research question.

## Running (reviewer reproduction)

```
docker build -t bbr3-kernel-replay .
docker run --rm --privileged \
  --sysctl net.ipv4.ip_forward=1 \
  -v $(pwd)/runs:/app/runs \
  bbr3-kernel-replay
```

Exit code 2 means `mahimahi` is not available in the container
(host doesn't support namespaces), and the sweep emitted a stub
CSV. This is the expected fallback on hosts without NET_ADMIN.

## Why a container?

The host kernel's BBR implementation is BBRv2; BBRv3 is in an
out-of-tree Google patch. Running the patched kernel without
reboot requires either a custom-built module or a nested QEMU VM;
the container isolates both approaches from the host.

## Known limitations

1. Requires NET_ADMIN on the host for mahimahi's namespace
   operations. Public-cloud containers without privileged
   mode will fall through to the stub path.
2. `ss -i`'s `pacing_rate` is reported in discrete units; very
   small values can be noisy. The production run uses an eBPF
   kprobe on `bbr_update_pacing_rate` instead for sub-RTT
   precision.
3. The paper's $T(B, D) = (\lfloor B/D \rfloor - 2)\cdot W + c$
   assumes $W$ in units of ticks; we pick tick = 1ms here, but
   kernel BBRv3 uses RTT-scaled ticks. The residual computation
   converts both to milliseconds for the comparison.
